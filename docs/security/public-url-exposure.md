# 공개 URL 오남용 점검 — 기준·평가·판정

로그인 없이 접근되는 URL 표면이 오남용(무단 조회·추측·열거·과다 소비)될 수 있는지 점검한 기록. 인상이 아니라 **기준을 세우고 표면별로 평가해 판정**한다.

> **범위**: 인증 게이트 밖이거나, 토큰 같은 capability만으로 접근되는 표면 — ① 인증 미디어 프록시 `/api/media`, ② 공개 grass 임베드 `/api/grass/[token]`. (그 외 페이지·액션은 미들웨어가 로그인을 강제하므로 제외.)

## 평가 기준

| 기준 | 묻는 것 |
|---|---|
| **접근 게이트** | 무엇이 접근을 막나 — 세션 인증 / RLS / 토큰(capability) / 없음 |
| **추측·열거 가능성** | URL/토큰을 추측하거나 순차 열거해 남의 자원에 닿을 수 있나 |
| **노출 범위** | 뚫렸을 때 무엇이 새나 — 원본 파일 / 집계만 / 메타데이터 |
| **검증 일치** | 입력 검증이 실제 생성 규칙과 맞나(느슨하면 약한 형태를 허용) |
| **소비(비용) 통제** | 반복 호출로 비용을 끌어낼 수 있나, 막는 장치가 있나 |

## 표면별 평가

### ① `/api/media/[...path]` — 인증 미디어 프록시

| 기준 | 평가 |
|---|---|
| 접근 게이트 | **호출자 세션으로** Storage download → Storage RLS `media: read own`이 실제 게이트. 익명·비소유자는 download 실패 → **404** |
| 추측·열거 | 경로를 알아도 **세션이 소유자가 아니면 RLS가 거부**. 추측이 무의미 |
| 노출 범위 | 통과해도 **본인 파일만**. 무기한 signed-URL을 뿌리지 않음(프록시가 매 요청 인가) |
| 검증 일치 | 경로는 Storage RLS가 `(foldername)[1] = auth.uid()`로 강제 |
| 소비 통제 | 인증 필요 + (별도) egress는 CDN 단계 과제 |

**판정: 안전 — 조치 불요.** capability가 아니라 *세션 인가*에 매번 의존하는 구조라, URL이 유출돼도 소유자 세션 없이는 못 연다.

### ② `/api/grass/[token]` — 공개 grass 임베드

| 기준 | 평가 |
|---|---|
| 접근 게이트 | **토큰(capability)** — 임베드용이라 의도적으로 비로그인 공개. `enabled=false`면 무효 |
| 추측·열거 | 토큰은 `gen_random_bytes(12)` = **96비트(24 hex)** → 추측·열거 비현실적 |
| 노출 범위 | `get_grass`가 **집계(일자별 카운트)만** 반환 — 원본·민감정보 비노출 |
| 검증 일치 | ⚠️ 정규식이 `[a-f0-9]{8,64}` 로 **생성물(24자)보다 느슨** — 8자(32비트) 형태도 받아들임 |
| 소비 통제 | rate limit **IP 60/분**(토큰 검증 전 적용) + 응답 `Cache-Control` 600s |

**판정: 대체로 안전, 검증 일치만 하드닝.** 토큰 강도·노출 범위·소비 통제는 충분하나, **검증이 생성 규칙보다 느슨한 mismatch**는 고친다 — 입력 검증은 실제 형식보다 좁아야 하며(느슨하면 약한 토큰 형태나 무효 입력을 더 받음), 생성이 항상 24 hex이므로 검증도 정확히 24 hex로 좁힌다.

## 적용한 조치

- **grass 토큰 검증 강화**: `/^[a-f0-9]{8,64}$/` → **`/^[a-f0-9]{24}$/`** (생성 규칙 `gen_random_bytes(12)`와 일치). 정상 토큰은 모두 24자라 영향 없음. 무효·짧은 토큰은 더 이르게 404로 차단되고, brute-force가 노릴 키공간 가정도 실제(96비트)로 고정된다.
- media 프록시·토큰 강도·노출 범위는 이미 충분 → 변경 없음.

## 잔여 위험 / 후속

- **토큰 회전(rotate)** 기능은 없음 — 유출 시 현재는 `enabled=false`로 무효화하거나 tracker를 재생성해야 한다. 명시적 rotate는 후속 과제.
- grass는 **capability URL** 특성상 "토큰을 아는 누구나 집계 조회 가능"이 의도된 동작 — 민감정보가 아니라 활동 집계라 수용 가능. 민감해지면 서명·만료(signed) 방식으로 전환 검토.
